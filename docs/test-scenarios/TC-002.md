# TC-002 - Create a marketplace offer and verify it in My Offers

**Tags:** `@TC-002` `@UI`

## Test objective

Verify that a logged-in user can create a new sell offer for one of their own items on the marketplace, and that the created offer appears on their own offers list ("My Offers").

## Preconditions

- Environment: local (`rolnopol` running on `http://localhost:3000`)
- User: a new account created via the API, with one animal added to it via the API (an offer can only be created for an item the user already owns), logged in by injecting the session directly (without going through the UI login flow)
- Test data: offer price and description generated by faker

## Test steps

| Step | Action                                                             | Expected result                                                  |
| ---- | ------------------------------------------------------------------ | ---------------------------------------------------------------- |
| 1    | Navigate to `/marketplace.html` with an injected session           | Marketplace page is visible, user is logged in                   |
| 2    | Go to the "Create Offer" tab                                       | The offer creation form is visible                               |
| 3    | Select the owned animal as the item, fill in price and description | Values are entered into the form fields                          |
| 4    | Click the "Create Offer" button                                    | Offer is created, confirmation is visible                        |
| 5    | Go to the "My Offers" tab                                          | The newly created offer is visible in the user's own offers list |

## Final result

A new offer is successfully created for the user's own animal and visible in the "My Offers" tab.

## Technical notes

- Cleanup order matters: the created offer must be cancelled, then the animal removed, then the account deleted - none of these cascade automatically.
