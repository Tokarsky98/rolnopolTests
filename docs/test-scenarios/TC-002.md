# TC-002 - Create a marketplace offer and verify it in My Offers

**Tags:** `@TC-002` `@UI`

## Test objective

Verify that a logged-in user can create a new sell offer on the marketplace, and that the created offer appears on their own offers list ("My Offers").

## Preconditions

- Environment: local (`rolnopol` running on `http://localhost:3000`)
- User: a new account created via the API (`POST /api/register`, faker data), logged in by injecting the session directly (without going through the UI login flow)
- Test data: offer data generated by faker (product name, quantity, price, description)

## Test steps

| Step | Action                                                      | Expected result                                                  |
| ---- | ----------------------------------------------------------- | ---------------------------------------------------------------- |
| 1    | Navigate to `/marketplace.html` with an injected session    | Marketplace page is visible, user is logged in                   |
| 2    | Go to the "Create Offer" tab                                | The offer creation form (`#createOfferForm`) is visible          |
| 3    | Fill in the offer form (name, quantity, price, description) | Values are entered into the form fields                          |
| 4    | Click the "Create Offer" button                             | Offer is created, confirmation is visible                        |
| 5    | Go to the "My Offers" tab                                   | The newly created offer is visible in the user's own offers list |

## Final result

A new offer is successfully created and visible in the "My Offers" tab of the logged-in user.

## Technical notes

- Cleanup order matters: the created offer must be deleted before the account, since account deletion does not cascade to related resources.
