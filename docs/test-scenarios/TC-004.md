# TC-004 - Registration rejects a duplicate email

**Tags:** `@TC-004` `@API`

## Test objective

Verify that attempting to register a new account with an email that is already in use is rejected with a clear error.

## Preconditions

- Environment: local (`rolnopol` running on `http://localhost:3000`)
- User: none logged in
- Test data: a freshly generated email/password/display name (faker), reused for a second registration attempt

## Test steps

| Step | Action                                                | Expected result                               |
| ---- | ----------------------------------------------------- | --------------------------------------------- |
| 1    | Register a new account with a freshly generated email | Account is created successfully               |
| 2    | Attempt to register again using the same email        | Request is rejected                           |
| 3    | Inspect the error response                            | Response indicates the account already exists |

## Final result

The second registration attempt fails with a client error and a message indicating the email is already in use.

## Technical notes

- Only the account created in step 1 exists and is deleted after the test, the rejected second attempt creates no account.
