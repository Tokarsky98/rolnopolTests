# TC-003 - Marketplace offers listing returns valid data

**Tags:** `@TC-003` `@API`

## Test objective

Verify that the marketplace offers endpoint returns a successful response with correctly structured offer data.

## Preconditions

- Environment: local (`rolnopol` running on `http://localhost:3000`)
- User: an authenticated account with a valid session
- Test data: none required beyond an authenticated session

## Test steps

| Step | Action                                            | Expected result                                                                     |
| ---- | ------------------------------------------------- | ----------------------------------------------------------------------------------- |
| 1    | Send a request to the marketplace offers endpoint | Response status is 200                                                              |
| 2    | Inspect the response body                         | Response is a list of offers                                                        |
| 3    | Inspect each offer in the list                    | Each offer contains the required fields (id, product name, price, quantity, seller) |

## Final result

The endpoint returns a 200 response containing a well-formed list of marketplace offers with all required fields present.

## Technical notes

- Uses soft assertions so any missing or invalid fields across the whole list are reported at once instead of failing at the first one.
