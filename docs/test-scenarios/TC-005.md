# TC-005 - Marketplace page shows an error notification when the offers request fails

**Tags:** `@TC-005` `@INTEGRATION`

## Test objective

Verify that the marketplace page shows a user-facing error notification instead of silently failing when the offers request fails.

## Preconditions

- Environment: local (`rolnopol` running on `http://localhost:3000`)
- User: a logged-in user (session injected)
- Test data: none - the offers network request is intercepted and made to return a server error

## Test steps

| Step | Action                                                                              | Expected result                                         |
| ---- | ----------------------------------------------------------------------------------- | ------------------------------------------------------- |
| 1    | Arm an interception that makes the marketplace offers request return a server error | Interception is set up before navigation                |
| 2    | Navigate to the marketplace page                                                    | Page loads, the offers request fails as configured      |
| 3    | Observe the page                                                                    | An error notification is shown with the failure message |

## Final result

The user sees a clear error notification when marketplace data cannot be loaded, rather than a silent failure.

## Technical notes

- The request is fulfilled with a 500 response rather than aborted, since an aborted request produces no visible feedback in the app.
