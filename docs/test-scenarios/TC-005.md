# TC-005 - Marketplace page shows an error state when the offers request fails

**Tags:** `@TC-005` `@INTEGRATION`

## Test objective

Verify that the marketplace page shows a user-facing error state instead of silently failing when the offers request fails.

## Preconditions

- Environment: local (`rolnopol` running on `http://localhost:3000`)
- User: a logged-in user (session injected)
- Test data: none - the offers network request is intercepted and forced to fail

## Test steps

| Step | Action                                                                 | Expected result                                                   |
| ---- | ---------------------------------------------------------------------- | ----------------------------------------------------------------- |
| 1    | Arm an interception that forces the marketplace offers request to fail | Interception is set up before navigation                          |
| 2    | Navigate to the marketplace page                                       | Page loads, the offers request fails as configured                |
| 3    | Observe the offers section                                             | An error state/message is shown instead of a blank or broken list |

## Final result

The user sees a clear error indication when marketplace data cannot be loaded, rather than a silent failure or broken UI.

## Technical notes

- The network failure is simulated by intercepting the request, the real backend is never called for the failing request.
