# TC-006 - New user can purchase an item using deposited funds

**Tags:** `@TC-006` `@UI` `@API`

## Test objective

Verify that a funded user can buy another user's marketplace offer through the UI, and that the purchase is correctly reflected both in the buyer's balance and in the marketplace listing.

## Preconditions

- Environment: local (`rolnopol` running on `http://localhost:3000`)
- Users: two accounts created via the API - a seller with an active offer, and a buyer with funds deposited into their account (both created and funded via the API, ahead of the actual purchase step)
- Test data: offer data (faker) for the seller, a deposit amount for the buyer greater than the offer price

## Test steps

| Step | Action                                                   | Expected result                                                                     |
| ---- | -------------------------------------------------------- | ----------------------------------------------------------------------------------- |
| 1    | Log in as the buyer and navigate to the marketplace page | The seller's offer is visible with a "Buy Now" button                               |
| 2    | Click "Buy Now" on the seller's offer                    | A purchase confirmation dialog is shown                                             |
| 3    | Confirm the purchase in the dialog                       | Confirmation of a successful purchase is shown, offer is no longer available to buy |
| 4    | Fetch the buyer's balance via the API                    | Buyer's balance is reduced by the offer price                                       |

## Final result

The buyer successfully purchases the offer through the UI, the purchase is confirmed on screen, and the buyer's balance is correctly debited.

## Technical notes

- Seller, offer and buyer funding are all set up via the API before the test, only the purchase itself happens through the UI.
- Two accounts are involved and both are deleted after the test, the offer is cancelled first if it was not purchased during the test.
