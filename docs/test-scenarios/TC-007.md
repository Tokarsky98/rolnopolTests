# TC-007 - Adding a farm animal with an overridden animal type

**Tags:** `@TC-007` `@API`

## Test objective

Verify that a farm animal can be added to a user's account via the API, and that the default test fixture can be overridden to create a different animal type without changing the test body.

## Preconditions

- Environment: local (`rolnopol` running on `http://localhost:3000`)
- User: an account created via the API
- Test data: the fixture creates a default animal type unless overridden, this test overrides it with a different type

## Test steps

| Step | Action                                                          | Expected result                                                            |
| ---- | --------------------------------------------------------------- | -------------------------------------------------------------------------- |
| 1    | Override the default animal fixture with a specific animal type | The fixture creates that type of animal via the API instead of the default |
| 2    | Fetch the account's animals via the API                         | The overridden animal type is present in the list                          |
| 3    | Inspect the animal's details                                    | Animal data (type, amount) matches what was requested in the override      |

## Final result

The animal created for the account matches the type specified by the override, confirming the fixture can be reused with different data across tests.

## Technical notes

- The animal is deleted after the test regardless of which type was used, followed by deleting the account.
