# TC-001 - Newly registered user can log in

**Tags:** `@TC-001` `@UI`

## Test objective

Verify that a user registered moments earlier via the API can successfully log in through the real UI form and gets authenticated (sees a personalized greeting).

## Preconditions

- Environment: local (`rolnopol` running on `http://localhost:3000`)
- User: no active session in the browser (guest)
- Test data: a new account created right before the test via `POST /api/register` (email, display name, password generated by faker)

## Test steps

| Step | Action                                              | Expected result                                                        |
| ---- | --------------------------------------------------- | ---------------------------------------------------------------------- |
| 1    | Navigate to the login page `/login.html`            | Login form is visible                                                  |
| 2    | Enter the email and password of the created account | Values are entered into the form fields                                |
| 3    | Click the login button                              | User is redirected to the profile page                                 |
| 4    | Check the greeting in the navbar                    | Personalized greeting containing the user's `displayedName` is visible |

## Final result

The user is successfully logged in through the real UI form and sees a greeting matching the data provided at registration.

## Technical notes

- The account is created via the API before the test and deleted via the API after the test (self-cleanup).
